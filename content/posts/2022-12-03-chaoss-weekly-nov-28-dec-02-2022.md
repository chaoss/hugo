### <span style="color: #993300;">Reminder: Holiday Schedule for CHAOSS Meetings</span>

This year we are following our tradition of taking a little break from official CHAOSS meetings at the end of the year. Please note the following meeting cancellations, with particular attention to the App Ecosystem meetings:

  * **December 12- January 9 &#8211; no meetings**
  * **App Ecosystem Group &#8211; December 1-January 9 no meetings**

CHAOSS work can still occur asynchronously if an individual Working Group wishes, and a lot of us will be around in Slack periodically if anything urgent comes up. But the hope is that we all take this time to rest and protect ourselves from burnout, so that we can come back refreshed, rejuvenated and ready to start anew in 2023!\[/vc\_column\_text\]\[vc\_column\_text\]

### <span style="color: #993300;">CHAOSScon EU 2023 is Happening! </span>

We are super excited to announce that CHAOSScon EU 2023 is happening in conjunction with FOSDEM in Brussels, Belgium.  CHAOSScon day will happen on February 3 from 8:00 to 17:00 Central European Standard Time. Although we are still finalizing some details, **registration is open** and you can learn more about this conference on the [CHAOSScon EU 2023 website. ][1]

Hope you can join us!\[/vc\_column\_text\]\[vc\_column\_text\]

### <span style="color: #993300;">Next Onboarding Session for New Chaotics</span>

The CHAOSS community continues to grow! To make the onboarding process a little easier, Elizabeth Barron (CHAOSS Community Manager) and Ruth Ikegah (CHAOSS Africa Community Lead) will be hosting the next Onboarding Session on January 11, 2023 at 11:00 am US Central/Chicago time**.** We know this time is not convenient for everyone, but this is just a start and we will add and iterate as we go.

This meeting will cover:

  * General overview of CHAOSS
  * How you can begin contributing
  * How to get started with using the metrics
  * Q&A

We hope you can join to meet other newcomers and learn about CHAOSS! You can [convert to your time zone here][2].\[/vc\_column\_text\]\[vc\_column\_text\]

### <span style="color: #993300;">Major Augur Release! </span>

Huge congratulations to the whole Augur team for their latest (and arguably largest to-date) release! From Sean Goggins&#8217;s announcement in Slack, here is what you can expect to find in this latest release:

Augur is releasing a significant update following an extensive beta testing regimen today. Thank you to the 239 contributors to this release, especially the <span class="p-member_profile_hover_card" role="presentation"><a class="c-link c-member_slug c-member_slug--light c-member_slug--link" tabindex="-1" href="https://chaoss-workspace.slack.com/team/U03DJ8EBC5Q" rel="noopener noreferrer" data-member-id="U03DJ8EBC5Q" data-member-label="@welcome-to-CHAOSS" data-stringify-type="mention" data-stringify-id="U03DJ8EBC5Q" data-stringify-label="@welcome-to-CHAOSS" aria-hidden="false" data-remove-tab-index="true" data-tab-index-to-restore="0">@welcome-to-CHAOSS</a></span> app, the CHAOSS Community, <a class="c-link c-mrkdwn__channel c-member_slug internal_channel_link c-member_slug--link" tabindex="-1" href="https://chaoss-workspace.slack.com/archives/C03KC6ZERSR" rel="noopener noreferrer" data-channel-name="#chaoss-africa" data-channel-id="C03KC6ZERSR" data-qa="channel_link" data-stringify-type="channel" data-sk="tooltip_parent" data-remove-tab-index="true">#chaoss-africa</a>, and Augur’s core maintainer team. This is Augur’s 133rd formal release and by a wide margin, the most significant advancement in Augur’s history. We look forward to folks opening issues and sharing feedback as we work toward a hosted version of Augur in our next release later this year. <a class="c-link c-link--underline" tabindex="-1" href="https://github.com/chaoss/augur/releases/tag/v0.43.0" target="_blank" rel="noopener noreferrer" data-stringify-link="https://github.com/chaoss/augur/releases/tag/v0.43.0" data-sk="tooltip_parent" data-remove-tab-index="true">https://github.com/chaoss/augur/releases/tag/v0.43.0</a>\[If you want to jump right in, updated docker build/compose and bare metal installation instructions are available here\](<a class="c-link c-link--underline" tabindex="-1" href="https://github.com/chaoss/augur/docs/new-install.md" target="_blank" rel="noopener noreferrer" data-stringify-link="https://github.com/chaoss/augur/docs/new-install.md" data-sk="tooltip_parent" data-remove-tab-index="true">https://github.com/chaoss/augur/docs/new-install.md</a>)Augur is now releasing a dramatically improved new version to the main branch. It is also available here: <a class="c-link c-link--underline" tabindex="-1" href="https://github.com/chaoss/augur/releases/tag/v0.43.0" target="_blank" rel="noopener noreferrer" data-stringify-link="https://github.com/chaoss/augur/releases/tag/v0.43.0" data-sk="tooltip_parent" data-remove-tab-index="true">https://github.com/chaoss/augur/releases/tag/v0.43.0</a>  
&#8211; The <code class="c-mrkdwn__code" data-stringify-type="code">main</code> branch is a stable version of our new architecture, which features:  
&#8211; Dramatic improvement in the speed of large-scale data collection (10,000+ repos). All data is obtained for 10k+ repos within a week  
&#8211; A new job management architecture that uses Celery and Redis to manage queues, and enables users to run a Flower job monitoring dashboard  
&#8211; Materialized views to increase the snappiness of APIs and Frontends on large-scale data  
&#8211; Changes to primary keys, which now employ a UUID strategy that ensures unique keys across all Augur instances  
&#8211; Support for <a class="c-link c-link--underline" tabindex="-1" href="https://github.com/chaoss/sandiego-rh" target="_blank" rel="noopener noreferrer" data-stringify-link="https://github.com/chaoss/sandiego-rh" data-sk="tooltip_parent" data-remove-tab-index="true">https://github.com/chaoss/sandiego-rh</a> dashboards (view a sample here: <a class="c-link c-link--underline" tabindex="-1" href="https://eightknot.osci.io/" target="_blank" rel="noopener noreferrer" data-stringify-link="https://eightknot.osci.io/" data-sk="tooltip_parent" data-remove-tab-index="true">https://eightknot.osci.io/</a>). (beautification coming soon!)  
&#8211; Data collection completeness assurance enabled by a structured, relational data set that is easily compared with platform API Endpoints  
&#8211; The next release of the new version will include a hosted version of Augur where anyone can create an account and add repos “they care about”. Augur’s new frontend (<a class="c-link c-link--underline" tabindex="-1" href="https://github.com/augurlabs/augur_view" target="_blank" rel="noopener noreferrer" data-stringify-link="https://github.com/augurlabs/augur_view" data-sk="tooltip_parent" data-remove-tab-index="true">https://github.com/augurlabs/augur_view</a>) will move to the CHAOSS org and be released with integration to the \*EightKnot\* project, providing advanced, dynamic visualizations of metrics, metrics models, and login functions. Users will have tight control of the repositories they see in views they create. The hosted instance will add repositories to a user’s view if it already has data for that requested organization or repository. If it’s a new repository or organization, the user will be notified that collection will take (time required for the scale of repositories added).

We are so proud of and thankful for all who contributed to this latest release!  Amazing work!\[/vc\_column\_text\]\[vc\_column\_text\]
